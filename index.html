Azure의 App Service 쓸 때 참고사항

  • Posted on July 21, 2018
  • by Sangyong Gwak

javascript 라이브러리를 이용하여 데이터를 받아왔는데, __createdAt, __updatedAt 컬럼의 값을 받아오질 않는 경우 SystemProperties.All 이 부분을 참고하세요.

var client = new WindowsAzure.MobileServiceClient('https://imyaman.azure-mobile.net/', 'MY-KEY');
var commentsItemTable = client.getTable('Comments');
commentsItemTable.systemProperties = WindowsAzure.MobileServiceTable.SystemProperties.All;

Azure에서 ZUMO-API-VERSION이 포함된 메시지를 보면, 원인을 추측하지 말고 바로 검색하는 것이 좋습니다.
앱 세팅에 MS_SkipVersionCheck = true 설정을 하면 해결됩니다.

App Service 상세 메뉴 중에 Overview 아래 쪽에 API -> CORS가 있습니다. Cross origin 관련해서 설정하면 편합니다.
cli로 설정해야할 땐 properties.cors.allowedOrigins 키워드로 찾아보세요.

포트 포워딩 예시

  • Posted on July 15, 2018
  • by Sangyong Gwak

사설 네트워크에 여러개의 서버가 있고, 그 중 하나의 서버에만 공인 IP가 있는 상황입니다. 외부에서 공인 IP를 통해 사설 네트워크 안의 서버에 접근하고 싶을 때, 포트 포워딩을 이용할 수 있습니다.

공인 IP가 있는 서버를 P1, 사설 네트워크 내부의 서버를 S1, S2라고 하겠습니다.

iptables로 아래와 같이 하면, P1의 1022 포트로 접근하면, S1의 22번 포트로 연결됩니다. S1의 주소가 10.1.1.3 입니다.

iptables -A PREROUTING -i ens3 -p tcp -m tcp --dport 1022 -j DNAT --to-destination 10.1.1.3:22
iptables -A PREROUTING -p tcp -m tcp --dport 1022 -j DNAT --to-destination 10.1.1.3:22
iptables -A POSTROUTING -o ens3 -j MASQUERADE
iptables -A FORWARD -p tcp -m tcp --dport 22 -j ACCEPT
iptables -A FORWARD -p tcp -m tcp --dport 1022 -j ACCEPT

저는 이렇게 설정파일을 만들어 쓰고 있습니다.
