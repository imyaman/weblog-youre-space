GCP UPDATES  |  MARCH 4, 2019
Key announcements
DATA ANALYTICS
BigQuery sandbox: beta
Explore analytics in BigQuery at no cost. BigQuery sandbox lets you access the same compute power as paying users, with up to 1 terabyte per month of query capacity and 10 GB of free storage. Blog
Additional releases
COMPUTE
Kubernetes Engine – node auto-repair: GA
This feature makes periodic checks on the health state of each node in your cluster. If a node fails consecutive health checks over an extended time period, Kubernetes Engine initiates a repair process for that node. Documentation
Kubernetes Engine – node auto-upgrades: GA
Node auto-upgrades help you keep the nodes in your cluster up to date with the latest stable version of Kubernetes. Auto-upgrades use the same update mechanism as manual node upgrades. Documentation
Compute Engine – guest attributes: beta
Now you can allow applications or scripts running in guest environments to publish information to the metadata server, which was previously read-only for guests. You can use guest attributes to get more reliable status information, such as VM initialization, completion of operations, and more. Documentation
Kubernetes Engine – Application-layer Secrets Encryption: beta
This release allows you to apply an additional layer of security at the application level to protect sensitive data, such as Secrets, using a key that you manage in Cloud Key Management Service. Documentation
Compute Engine – health checking and autohealing for managed instance groups: GA
You can improve the availability of an application that’s running on a managed instance group by configuring health checking, which periodically verifies that the application is responding as expected on each VM instance in the group. If an application fails health checking, autohealing re-creates that instance from the original instance template to put it back in service. Documentation
DATA ANALYTICS
Cloud Composer – support for multiple Apache Airflow versions: GA
This update provides support for Apache Airflow versions 1.9.0 and 1.10.0 on Cloud Composer. You can choose which version of Airflow you want when creating a new Cloud Composer environment without enabling beta feature support. Documentation
BigQuery available in São Paulo: GA
With regional availability in southamerica-east1, you can now use BigQuery in the same local Google Cloud regions you already use for storage, compute, and machine learning. Documentation
AI & MACHINE LEARNING
Cloud TPU on Cloud Machine Learning Engine: GA
Now you can use Cloud TPU to accelerate workloads and train your models on Cloud ML Engine. And Cloud ML Engine provides a job management interface so that you don’t need to manage the Tensor Processing Unit yourself. Documentation
Compute Engine machine types: beta
This feature lets you configure your training jobs using Compute Engine predefined machine types. This gives you more flexibility when allocating computing resources for your training job, and it allows you to customize how your job uses graphics processing units. Documentation
Cloud ML Engine – training with custom containers: beta
Now you can use your own Docker containers with any pre-installed ML framework or algorithm to run on Cloud ML Engine. Documentation
STORAGE
Cloud Storage – Bucket Policy Only: beta
This feature lets you enforce Cloud Identity and Access Management policies at the bucket level, for uniform access to buckets. It allows access only through IAM policies, rather than through object-level access control lists. This feature also includes Cloud IAM security features.
 Documentation
MIGRATION
Transfer Appliance model TA480 available in Europe: beta
You can use this rackable high-capacity storage server to securely migrate large volumes of data – up to 1 petabyte – to Google Cloud Platform without disrupting your business operations. The larger model, TA480, is now available in Europe. Documentation
