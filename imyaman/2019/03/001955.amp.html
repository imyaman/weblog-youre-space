GCP UPDATES  |  FEBRUARY 19, 2019
Key announcements
DATABASES
Cloud Firestore: GA
Cloud Firestore is a fast, fully managed, serverless, cloud-native NoSQL document database. It simplifies storing, syncing, and querying data for your mobile, web, and IoT apps at global scale. Cloud Firestore is the next generation of Cloud Datastore. Product page | Documentation
Additional releases
COMPUTE
App Engine flexible environment – WebSockets support: beta
Achieve lower latency by using WebSockets to create persistent connections between clients and App Engine instances. For clients that don’t support WebSockets, you can enable session affinity, which routes App Engine requests by the same user to the same instance. Documentation
NETWORKING
Internal load balancer – service discovery: GA
Now you can assign a service label to the forwarding rule for an internal load balancer. This allows client VM instances in the same project, VPC network, and region to use the internal DNS name instead of the forwarding rule’s IP address to send traffic to the backend VMs. Documentation
Private Google Access for on-premises hosts: GA
This update allows you to connect your on-premises hosts to Google APIs and services through a Cloud VPN tunnel or Cloud Interconnect. Documentation
API MANAGEMENT
Apigee Developer Portal – SmartDocs, account management, and new theming tools: GA
These additions to the Apigee Developer Portal can help you create better developer experiences. Along with enhanced account management and theming features, this release includes SmartDocs for Apigee Developer Portal, which provides API documentation in a three-pane view. Blog | Product page
DEVELOPER TOOLS
SSH access to Cloud Source Repositories: beta
Cloud Source Repositories now allows you to use Secure Shell public key authentication to access hosted repositories. You first generate a key pair on the local system from which you want to access the repository, then register the public key with Google Cloud.
 Documentation
DATA ANALYTICS
BigQuery INFORMATION_SCHEMA: beta
INFORMATION_SCHEMA is a series of views – virtual tables defined by SQL queries – that provide access to metadata about datasets, tables, and views. Documentation
BigQuery – Cloud Audit Logs v2: beta
A new format of BigQuery log messages, BigQueryAuditMetadata, provides details on how resources are changed indirectly by other resources, particularly when reporting on asynchronous events and events that are not strongly coupled to a specific API call. Documentation
BigQuery available in two new locations: GA
Now you can specify dataset locations for BigQuery in Los Angeles (us-west2) and Hong Kong (asia-east2).
 Documentation
AI & MACHINE LEARNING
Cloud Machine Learning Engine – quad-core instances for online prediction: beta
This update makes quad-core CPUs available for the Cloud ML Engine online prediction service, which manages computing resources in the cloud to run your models. You can request predictions from your models and get predicted target values for them faster with more powerful CPUs. Documentation
Compute Engine – Cloud TPU v3: GA
Cloud TPU v3 improves performance, with 420 teraflops of computing speed and 128 GB of high-bandwidth memory. It supports more classes of models, such as deeper ResNets and larger images with RetinaNet. Product page | Documentation
